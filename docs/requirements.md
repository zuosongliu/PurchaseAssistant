## 1. 项目背景与目标

### 1.1 背景

在许多中大型企业中，采购申请流程存在以下痛点：

- 员工不了解采购规范，填报质量低，导致反复退回
- 审批人花费大量时间审阅低质量申请，效率低下
- 价格估算缺乏参考依据，申请金额随意性强
- 采购管理层缺乏数据洞察，难以发现异常或优化空间

### 1.2 目标

本项目旨在基于 SAP BTP 构建 **AI 辅助采购平台**，实现以下目标：

| 目标 | 衡量指标 |
|------|----------|
| 提升采购申请质量 | AI 质量评分平均分 ≥ 80 |
| 降低审批人工作量 | 审批平均处理时长减少 ≥ 30% |
| 提高合规性 | 违规申请比例降低 ≥ 50% |
| 提供管理洞察 | 支持按部门/品类/供应商的多维度分析 |

### 1.3 范围

- **包含：** 采购申请的创建、提交、审批、分析全流程，以及相关主数据维护
- **不包含：** 采购订单（PO）创建、收货、发票处理等后续流程
- **平台：** SAP BTP (Cloud Foundry)，面向 Demo/POC 场景构建

## 2. 用户角色

### 2.1 员工（Employee / Requester）

**职责：** 创建并提交采购申请

**主要挑战：**
- 不清楚采购规范（需要哪些字段、供应商选谁、价格怎么估）
- 担心因信息不完整或不规范导致申请被退回
- 希望快速完成申请，减少反复修改

**关键需求：**
- 能用自然语言描述需求，由 AI 自动填充申请详情
- 获得实时的合规提示和质量反馈
- 查看自己提交的所有申请及其状态

### 2.2 经理（Manager / Approver）

**职责：** 审批下属提交的采购申请

**主要挑战：**
- 每天需要处理多条申请，时间有限
- 难以快速判断申请的合理性和风险
- 对采购政策细节掌握程度参差不齐

**关键需求：**
- AI 生成审批摘要，快速了解申请要点
- 风险评估和质量评分辅助判断
- 支持批量处理，减少重复操作

### 2.3 采购管理员（Procurement Administrator）

**职责：** 管理供应商、品类规则、定价参考；分析采购数据

**主要挑战：**
- 需要及时发现采购异常（集中采购、超预算、非合规供应商）
- 缺乏直观的数据可视化工具

**关键需求：**
- 多维度采购数据分析（部门、品类、供应商、时间）
- AI 自动生成文字洞察，说明异常和趋势
- 维护供应商名单、品类规则和定价参考区间

### 2.4 系统管理员（System Administrator）

**职责：** 管理系统配置、用户权限、AI 参数

**关键需求：**
- 配置审批权限阈值（如单笔审批上限）
- 维护 AI 规则参数（质量评分权重、风险阈值）
- 监控系统运行状态

## 4. 应用功能需求

### 4.1 My Purchase Requests（我的采购申请）

#### 概述

员工管理自己采购申请的主界面，采用 SAP Fiori Elements List Report + Object Page 模式。

#### 功能需求

**FR-01 申请列表展示**
- 显示当前用户的所有采购申请
- 支持按状态过滤（草稿/已提交/待审批/已批准/已驳回）
- 支持关键字搜索（标题、品类）
- 列表字段：申请编号、标题、品类、金额、提交日期、状态、AI 质量评分
- 申请编号支持点击跳转至详情页

**FR-02 KPI 摘要栏**
- 页面顶部展示：申请总数、待审批数量、YTD 总金额、平均 AI 质量评分

**FR-03 新建申请**
- 提供"New Request"按钮，跳转至标准申请表单
- 提供"AI-Assisted Request"按钮，打开 AI 辅助申请对话框（见 4.2）

**FR-04 申请状态**

| 状态 | 说明 |
|------|------|
| Draft（草稿） | 已创建但未提交 |
| Submitted（已提交） | 已提交，等待系统分配审批人 |
| Pending Approval（待审批） | 审批人已收到，等待处理 |
| Approved（已批准） | 审批通过 |
| Rejected（已驳回） | 审批驳回，可修改后重新提交 |

**FR-05 AI 质量评分展示**
- 在列表中以色块显示分值（绿色 ≥85 / 橙色 70–84 / 红色 < 70）
- 草稿状态申请显示"—"（尚未评分）

### 4.2 AI-Assisted Purchase Request（AI 辅助创建申请）

#### 概述

通过对话式界面，员工用自然语言描述采购需求，AI 自动生成完整的采购申请草稿，员工确认后添加到申请列表。

#### 功能需求

**FR-06 对话框布局**
- 弹出全屏对话框，左右两栏布局
- 左侧：对话区（AI 消息 + 用户输入框）
- 右侧：AI 生成的采购申请建议详情

**FR-07 自然语言输入**
- 用户在输入框中描述需求，点击发送
- 支持的输入示例：
  - "需要 5 台高性能开发用笔记本，预算每台 1.2 万左右"
  - "给市场部买一批 Q2 的办公耗材"
  - "续签 Adobe Creative Cloud 年度许可证"

**FR-08 AI 处理过程展示**
- 发送后显示进度提示（分步展示处理阶段）
- 阶段包括：理解需求 → 匹配品类与供应商 → 检查定价与合规 → 生成申请草稿

**FR-09 AI 建议内容（右侧面板）**
生成完成后，右侧面板展示以下内容：

| 字段 | 说明 |
|------|------|
| 申请标题 | AI 自动命名 |
| 品类 | 匹配到的采购品类 |
| 数量 | 从描述中提取 |
| 估价（单价/总价） | 基于市场参考价生成 |
| 推荐供应商 | 来自批准供应商名单 |
| 技术规格 | AI 生成的标准规格描述 |
| 业务理由 | AI 生成的理由文本（可编辑） |
| 合规标签 | 价格是否合理、供应商是否合规等 |
| AI 质量预估分 | 当前草稿的预估质量分 |

**FR-10 确认添加到列表**
- 用户点击"Confirm & Add to List"，申请以草稿状态加入列表
- 对话框关闭，列表顶部出现新记录
- 用户可在列表中进一步编辑后提交

**FR-11 多轮对话**
- 用户可追问或修改需求，AI 更新右侧建议内容
- 支持的追问示例："换成联想品牌"、"预算调整到 8000 以内"

### 4.3 Purchase Approval Inbox（采购审批收件箱）

#### 概述

经理的审批工作台，采用主从布局（左侧申请列表 + 右侧审批详情）。AI 功能辅助经理快速做出决策。

#### 功能需求

**FR-12 待审批列表**
- 显示所有待当前用户审批的申请
- 列表摘要信息：申请编号、标题、申请人、金额、提交日期、AI 质量分、风险等级
- 支持按风险等级、金额排序
- 高优先级申请（High Risk 或超期）标红点提示

**FR-13 申请详情展示**
点击列表项后，右侧展示完整信息：
- 基本信息（申请编号、申请人、部门、成本中心）
- 采购详情（品类、数量、单价、总价、规格、供应商、所需日期）
- 业务理由
- AI 审批摘要（见 FR-14）
- AI 质量评分（见 FR-15）
- 风险评估（见 FR-16）

**FR-14 AI 审批摘要（AI Approval Assistant）**
- 展示 AI 生成的自然语言摘要，概述申请要点、合规状态
- 显示审批推荐结论（建议批准 / 建议驳回 / 需进一步审查）
- 以彩色标签形式展示关键发现（如"价格在合理区间"、"预算使用率偏高"）

**FR-15 AI 质量评分**
- 展示总分（0–100）及四个维度的分项得分：完整性、合规性、价格合理性、理由质量
- 以进度条形式展示各维度分值

**FR-16 风险评估**
- 逐项列出风险检查结果，每项显示风险等级（Low / Medium / High）
- 检查项包括：预算可用性、供应商合规、预算消耗率、采购政策符合性
- 显示整体风险等级

**FR-17 审批操作**
- 提供"Approve"和"Reject"按钮
- 驳回时必须填写审批意见
- 批准时审批意见为可选
- 操作完成后从待办列表中移除，进入已审批记录

### 4.4 Procurement Analytics Dashboard（采购分析仪表板）

#### 概述

为采购管理员提供数据分析视图，支持多维度查看采购数据，并通过 AI 生成文字洞察。

#### 功能需求

**FR-18 KPI 卡片**
展示关键指标：
- YTD 总采购金额及同比变化
- 已处理申请数量
- 平均审批时长（天）
- AI 质量平均分

**FR-19 月度支出趋势图**
- 折线图展示近 6–12 个月实际支出与预算对比
- 支持按财年/季度切换

**FR-20 部门支出分布**
- 水平条形图展示各部门 YTD 采购金额排名
- 支持下钻查看部门内品类分布

**FR-21 品类分布**
- 环形图展示当季各品类支出占比

**FR-22 供应商排名**
- 列表展示 Top 供应商及其：YTD 金额、同比变化、综合评级

**FR-23 申请量热力矩阵**
- 二维热力图，横轴为品类，纵轴为部门
- 色深表示申请数量，快速识别采购集中区域

**FR-24 AI 文字洞察（AI Insights）**
- 自动生成 3–5 条文字洞察，类型包括：
  - **趋势（Trend）：** 某部门/品类支出显著增长或下降
  - **异常（Anomaly）：** 异常采购行为（集中采购、非合规供应商使用增加）
  - **机会（Opportunity）：** 跨部门合并采购的节省空间
- 每条洞察包含具体数字支撑

**FR-25 过滤器**
- 支持按财年/季度、部门过滤所有图表数据

### 4.5 Master Data Maintenance（主数据维护）

#### 概述

供采购管理员和系统管理员维护系统运行所需的主数据。

#### 功能需求

**FR-26 供应商管理**
- 维护批准供应商名单（新增、编辑、停用）
- 字段：供应商名称、品类、联系方式、资质状态、评级、合同有效期

**FR-27 品类规则管理**
- 配置各品类的采购规则（哪些成本中心可使用、是否需要额外审批）

**FR-28 定价参考区间**
- 为主要品类维护市场参考价区间（最低价、中位价、最高价）
- 用于 AI 质量评分中的价格合理性判断

**FR-29 AI 规则阈值配置**
- 审批权限金额阈值（如单笔 ≤10 万由直线经理审批，>10 万需部门总监）
- 质量评分各维度权重
- 风险评估的预算使用率阈值（如 75% = Medium，90% = High）

## 5. AI 功能需求

### 5.1 AI 申请建议生成

| 需求项 | 说明 |
|--------|------|
| **输入** | 用户自然语言描述 |
| **输出** | 结构化申请草稿（标题、品类、数量、规格、供应商、估价、理由） |
| **技术** | SAP Generative AI Hub（LLM） |
| **响应时间** | ≤ 5 秒（含进度动画） |
| **可覆盖性** | 用户可修改任何 AI 生成字段 |

### 5.2 AI 质量评分

| 需求项 | 说明 |
|--------|------|
| **输入** | 完整申请字段 + 主数据（供应商名单、定价参考、品类规则） |
| **输出** | 总分（0–100）+ 四维度分项分 |
| **技术** | CAP 规则引擎（主） + 可选 ML 模型（辅） |
| **触发时机** | 申请提交时异步计算，审批前完成 |
| **展示位置** | 申请列表（总分）、申请详情（总分+分项）、审批详情（总分+分项） |

### 5.3 AI 审批摘要

| 需求项 | 说明 |
|--------|------|
| **输入** | 申请详情 + 预算数据 + 供应商资质 + 历史审批记录 |
| **输出** | 自然语言摘要段落 + 推荐结论 + 关键发现标签数组 |
| **技术** | SAP Generative AI Hub（LLM） |
| **响应时间** | ≤ 3 秒（申请打开时异步加载） |
| **缓存策略** | 摘要在审批过程中缓存，申请数据变更后失效 |

### 5.4 风险评估

| 需求项 | 说明 |
|--------|------|
| **输入** | 申请数据 + 预算数据 + 供应商主数据 + 采购政策配置 |
| **输出** | 风险项列表（每项含等级和说明）+ 整体风险等级 |
| **技术** | CAP 规则引擎（无需 LLM） |
| **可配置性** | 所有阈值通过 Master Data Maintenance 配置，无需修改代码 |

### 5.5 AI 分析洞察

| 需求项 | 说明 |
|--------|------|
| **输入** | 聚合后的采购统计数据（近期趋势、部门分布、供应商行为） |
| **输出** | 3–5 条分类洞察文本（趋势/异常/机会） |
| **技术** | SAP Generative AI Hub（LLM） |
| **刷新频率** | 每日定时生成，支持手动刷新 |

### 5.6 通用 AI 原则

- **AI 辅助，人工决策：** 所有 AI 输出为建议，不替代人工判断
- **可解释性：** 每个评分/风险项附带说明，用户可理解依据
- **可覆盖性：** 用户可修改或忽略任何 AI 生成内容
- **透明度：** 界面明确标注哪些内容由 AI 生成
- **数据安全：** 发送给 LLM 的数据需脱敏，不包含个人身份信息

## 6. 非功能性需求

### 6.1 性能

| 指标 | 要求 |
|------|------|
| 页面加载时间 | ≤ 3 秒（首屏） |
| AI 申请建议生成 | ≤ 5 秒 |
| AI 审批摘要加载 | ≤ 3 秒 |
| 质量评分/风险评估读取 | ≤ 1 秒（已缓存） |
| 分析仪表板加载 | ≤ 5 秒 |

### 6.2 安全与权限

| 角色 | 可访问应用 |
|------|----------|
| 员工 | My Purchase Requests、AI-Assisted Request |
| 经理 | 员工权限 + Purchase Approval Inbox |
| 采购管理员 | 经理权限 + Analytics Dashboard + Master Data Maintenance |
| 系统管理员 | 全部应用 |

- 员工只能查看和操作自己的申请
- 经理只能审批自己权限范围内的申请
- 基于 SAP BTP XSUAA 实现 OAuth 2.0 认证与角色授权

### 6.3 用户体验

- 遵循 SAP Fiori Design Guidelines
- 支持响应式布局（桌面端优先）
- AI 生成内容需有明显标识（✨ 图标或"AI Generated"标签）
- 所有操作提供及时的用户反馈（Toast、MessageStrip）

### 6.4 可靠性

- LLM 调用失败时，系统应降级处理（隐藏 AI 模块，展示静态内容）
- 数据操作（申请提交、审批）不依赖 AI 服务，AI 不可用不影响核心流程

## 7. 约束与假设

### 7.1 技术约束

- 后端：SAP CAP (Node.js)，部署于 SAP BTP Cloud Foundry
- 数据库：SAP HANA Cloud
- 前端：SAP Fiori Elements + SAPUI5 (sap_horizon 主题)
- AI：SAP Generative AI Hub（支持 Claude / GPT-4o 等模型）
- 认证：SAP BTP XSUAA

### 7.2 业务假设

- 本项目为 Demo/POC 阶段，数据为模拟数据，不对接真实 ERP 系统
- 预算数据、供应商数据、定价参考数据由演示用主数据提供
- 审批流程为单级审批（直线经理），暂不支持多级审批链
- 金额单位统一为人民币（CNY/¥）

### 7.3 依赖项

| 依赖 | 说明 |
